
# Release 1.2.2 (2026-03-01)

## Bug fixes

- **Ruby list DISPLAY methods without name_default use required positional name param (#158)**
The Ruby template incorrectly used optional keyword name: nil for all list DISPLAY methods. Only methods with name_default (display_queue, display_channel) should use optional keyword. All other list DISPLAY methods use a required positional name parameter, matching the existing hand-written Ruby code and RBS type signatures.


## CI

- **add concurrency group to ci-push workflow (#167)**
Ensures new pushes to the same branch cancel in-flight push CI runs, preventing stale checks from completing alongside newer ones.


## Documentation

- **move Releases nav to right of Home for consistency (#136)**
- **add multi-repo finalization workflow rules to CLAUDE.md (#156)**
- **add Python 3.12 to dev-python version matrix in CLAUDE.md (#166)**
CLAUDE.md listed only 3.13 and 3.14 but docker/build.sh and docker-publish.yml already include 3.12. Aligns documentation with the actual build matrix.


## Features

- **add Rust development tooling (#139)**
* feat: add Rust development tooling

Add Rust support: Docker dev image (rust:VERSION-slim with clippy, rustfmt, cargo-deny, cargo-llvm-cov), validate-local-rust script, Cargo.toml detection in docker-test, crates.io registry lookup, and build.sh entries for Rust 1.92/1.93.

- **add st-generate-commands CLI for multi-language MQSC method generation (#154)**
New CLI tool that reads mapping-data.json and generates complete MQSC command method blocks for Python, Ruby, Java, Go, and Rust. Supports marker-based file updates, --check mode for CI validation, and mapping pages for Python docstrings. All 4 command patterns (singleton, list, required_name, optional_name) are handled across all 5 languages.

- **add canonical label registry and sync modes to st-ensure-label (#164)**
Add labels.json registry with 10 canonical labels and enhancement delete list. Create lib/labels.py loader using importlib.resources. Enhance st-ensure-label with three modes: single-label (with --color/--description), --sync (provision all from registry), and --owner/--project (discover repos via GitHub Project). Add issue type dropdown to issue template. Package-data config added to pyproject.toml.
